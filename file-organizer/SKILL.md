# 文件整理器

此技能充当你的个人组织助手，帮助你维护干净、逻辑清晰的文件结构，无需持续手动组织的认知负担。

## 何时使用此技能

- 你的下载文件夹一团糟
- 你找不到文件因为它们散落各处
- 你有重复文件占用空间
- 你的文件夹结构不再有意义
- 你想建立更好的组织习惯
- 你正在开始新项目并需要好的结构
- 你正在归档旧项目前进行清理

## 此技能的功能

1. **分析当前结构**：审查你的文件夹和文件以了解你有什么
2. **查找重复文件**：识别你系统中的重复文件
3. **建议组织**：根据你的内容提出逻辑文件夹结构
4. **自动清理**：在你批准后移动、重命名和组织文件
5. **保持上下文**：基于文件类型、日期和内容做出智能决策
6. **减少混乱**：识别你可能不再需要的旧文件

## 说明

当用户请求文件组织帮助时：

1. **理解范围**
   
   提出澄清问题：
   - 哪个目录需要组织？（下载、文档、整个主文件夹？）
   - 主要问题是什么？（找不到东西、重复、太乱、没有结构？）
   - 有任何要避免的文件或文件夹吗？（当前项目、敏感数据？）
   - 组织的激进程度？（保守与全面清理）

2. **分析当前状态**
   
   审查目标目录：
   ```bash
   # 获取当前结构概览
   ls -la [target_directory]
   
   # 检查文件类型和大小
   find [target_directory] -type f -exec file {} \; | head -20
   
   # 识别最大文件
   du -sh [target_directory]/* | sort -rh | head -20
   
   # 统计文件类型
   find [target_directory] -type f | sed 's/.*\.//' | sort | uniq -c | sort -rn
   ```
   
   总结发现：
   - 文件和文件夹总数
   - 文件类型分解
   - 大小分布
   - 日期范围
   - 明显的组织问题

3. **识别组织模式**
   
   基于文件，确定逻辑分组：
   
   **按类型**：
   - 文档（PDF、DOCX、TXT）
   - 图像（JPG、PNG、SVG）
   - 视频（MP4、MOV）
   - 压缩包（ZIP、TAR、DMG）
   - 代码/项目（带代码的目录）
   - 电子表格（XLSX、CSV）
   - 演示文稿（PPTX、KEY）
   
   **按用途**：
   - 工作与个人
   - 活动与归档
   - 特定项目
   - 参考资料
   - 临时/暂存文件
   
   **按日期**：
   - 当前年/月
   - 过去年份
   - 非常旧（归档候选）

4. **查找重复文件**
   
   当请求时，搜索重复文件：
   ```bash
   # 通过哈希查找完全重复
   find [directory] -type f -exec md5 {} \; | sort | uniq -d
   
   # 查找同名文件
   find [directory] -type f -printf '%f\n' | sort | uniq -d
   
   # 查找相似大小的文件
   find [directory] -type f -printf '%s %p\n' | sort -n
   ```
   
   对于每组重复文件：
   - 显示所有文件路径
   - 显示大小和修改日期
   - 推荐保留哪个（通常是最新或命名最好的）
   - **重要**：删除前始终请求确认

5. **提出组织计划**
   
   在进行更改之前提出清晰的计划：
   
   ```markdown
   # [目录] 组织计划
   
   ## 当前状态
   - Y 个文件夹中的 X 个文件
   - 总 [大小]
   - 文件类型：[分解]
   - 问题：[列出问题]
   
   ## 建议结构
   
   [Directory]/
   ├── Work/
   │   ├── Projects/
   │   ├── Documents/
   │   └── Archive/
   ├── Personal/
   │   ├── Photos/
   │   ├── Documents/
   │   └── Media/
   └── Downloads/
       ├── To-Sort/
       └── Archive/
   
   ## 我将做的更改
   
   1. **创建新文件夹**：[列表]
   2. **移动文件**：
      - X 个 PDF → Work/Documents/
      - Y 个图像 → Personal/Photos/
      - Z 个旧文件 → Archive/
   3. **重命名文件**：[任何重命名模式]
   4. **删除**：[重复或垃圾文件]
   
   ## 需要你决定的文件
   
   - [列出任何不确定的文件]
   
   准备继续？（是/否/修改）
   ```

6. **执行组织**
   
   批准后，系统地组织：
   
   ```bash
   # 创建文件夹结构
   mkdir -p "path/to/new/folders"
   
   # 移动文件并带清晰日志
   mv "old/path/file.pdf" "new/path/file.pdf"
   
   # 使用一致模式重命名文件
   # 示例："YYYY-MM-DD - 描述.ext"
   ```
   
   **重要规则**：
   - 删除任何东西前始终确认
   - 记录所有移动以便潜在撤销
   - 保留原始修改日期
   - 优雅处理文件名冲突
   - 如果遇到意外情况，停止并询问

7. **提供摘要和维护提示**
   
   组织后：
   
   ```markdown
   # 组织完成！✨
   
   ## 更改内容
   
   - 创建了 [X] 个新文件夹
   - 组织了 [Y] 个文件
   - 通过删除重复文件释放了 [Z] GB
   - 归档了 [W] 个旧文件
   
   ## 新结构
   
   [显示新文件夹树]
   
   ## 维护提示
   
   保持组织：
   
   1. **每周**：整理新下载
   2. **每月**：审查并归档已完成的项目
   3. **每季度**：检查新重复文件
   4. **每年**：归档旧文件
   
   ## 快速命令
   
   ```bash
   # 查找本周修改的文件
   find . -type f -mtime -7
   
   # 按类型排序下载
   [为你的设置自定义命令]
   
   # 查找重复文件
   [自定义命令]
   ```
   
   想要整理另一个文件夹吗？
   ```

2. 找到模式：工作文档、个人照片、安装程序、随机 PDF

## 专业提示

1. **从小开始**：从一个混乱的文件夹开始（如下载）以建立信任
2. **定期维护**：每周在下载上运行清理
3. **一致命名**：对重要文件使用"YYYY-MM-DD - 描述"格式
4. **积极归档**：将旧项目移动到 Archive 而非删除
5. **保持活动分离**：维持活动和归档工作之间的清晰边界
6. **信任过程**：让 Claude 处理东西去哪里的认知负担

## 最佳实践

### 文件夹命名
- 使用清晰、描述性的名称
- 避免空格（使用连字符或下划线）
- 要具体："client-proposals"而非"docs"
- 使用前缀排序："01-current"、"02-archive"

### 文件命名
- 包括日期："2024-10-17-meeting-notes.md"
- 要描述性："q3-financial-report.xlsx"
- 避免在名称中使用版本号（改用版本控制）
- 删除下载伪影："document-final-v2 (1).pdf" → "document.pdf"

### 何时归档
- 6 个月以上未触及的项目
- 完成后可能稍后参考的工作
- 迁移到新系统后的旧版本
- 你犹豫是否要删除的文件（先归档）
