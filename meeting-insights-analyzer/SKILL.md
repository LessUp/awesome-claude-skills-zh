# 会议洞察分析器

此技能将你的会议转录转化为关于你沟通模式的可操作洞察，帮助你成为更有效的沟通者和领导者。

## 何时使用此技能

- 分析你跨多次会议的沟通模式
- 获取关于你的领导和引导风格的反馈
- 识别你何时回避困难对话
- 了解你的说话习惯和填充词
- 随时间追踪沟通技能的改进
- 用具体示例准备绩效审查
- 指导团队成员改善他们的沟通风格

## 此技能的功能

1. **模式识别**：识别跨会议的重复行为，如：
   - 冲突回避或间接沟通
   - 说话比例和轮流模式
   - 提问与陈述模式
   - 积极倾听指标
   - 决策方法

2. **沟通分析**：评估沟通有效性：
   - 清晰度和直接性
   - 填充词和模糊语言的使用
   - 语气和情感模式
   - 会议控制和引导

3. **可操作反馈**：提供具体的、带时间戳的示例，包括：
   - 发生了什么
   - 为什么重要
   - 如何改进

4. **趋势追踪**：在分析多次会议时比较随时间的模式

## 如何使用

### 基础设置

1. 将会议转录下载到文件夹（如 `~/meetings/`）
2. 在 Claude Code 中导航到该文件夹
3. 请求你想要的分析

### 快速开始示例

```
分析此文件夹中的所有会议，告诉我我何时回避了冲突。
```

```
查看我过去一个月的会议并识别我的沟通模式。
```

```
比较这两个会议文件夹之间的我的引导风格。
```

### 高级分析

```
分析此文件夹中的所有转录并：
1. 识别我何时打断他人
2. 计算我的说话比例
3. 找到我避免给出直接反馈的时刻
4. 追踪我的填充词使用
5. 展示积极倾听的示例
```

## 说明

当用户请求会议分析时：

1. **发现可用数据**
   - 扫描文件夹中的转录文件（.txt、.md、.vtt、.srt、.docx）
   - 检查文件是否包含说话者标签和时间戳
   - 确认会议的日期范围
   - 在转录中识别用户的名字/标识符

2. **澄清分析目标**
   
   如果未指定，询问他们想学习什么：
   - 特定行为（冲突回避、打断、填充词）
   - 沟通有效性（清晰度、直接性、倾听）
   - 会议引导技能
   - 说话模式和比例
   - 需要改进的领域
   
3. **分析模式**

   对于每个请求的洞察：
   
   **冲突回避**：
   - 寻找模糊语言（"也许"、"有点"、"我认为"）
   - 间接措辞而非直接请求
   - 紧张出现时改变话题
   - 同意但不承诺（"是的，但是..."）
   - 不 addressing 明显问题
   
   **说话比例**：
   - 计算会议中说话的百分比
   - 计数打断（用户打断和被他人打断）
   - 测量平均说话轮次长度
   - 追踪问题与陈述比例
   
   **填充词**：
   - 计数"嗯"、"啊"、"像"、"你知道"、"实际上"等
   - 注意每分钟或每说话轮的频率
   - 识别它们增加的情况（紧张、不确定）
   
   **积极倾听**：
   - 引用他人先前观点的问题
   - 重述或总结他人的想法
   - 在他人贡献基础上构建
   - 提出澄清问题
   
   **领导与引导**：
   - 决策方法（指令式与协作式）
   - 如何处理分歧
   - 包含较安静的参与者
   - 时间管理和议程控制
   - 跟进和行动项清晰度

4. **提供具体示例**

   对于发现的每个模式，包括：
   
   ```markdown
   ### [模式名称]
   
   **发现**：[模式的一句话摘要]
   
   **频率**：[Y 次会议中 X 次]
   
   **示例**：
   
   1. **[会议名称/日期]** - [时间戳]
      
      **发生了什么**：
      > [转录中的实际引用]
      
      **为什么这很重要**：
      [影响或错过机会的解释]
      
      **更好的方法**：
      [具体的替代措辞或行为]
   
   [重复 2-3 个最强示例]
   ```

5. **综合洞察**

   在分析所有模式后，提供：
   
   ```markdown
   # 会议洞察摘要
   
   **分析周期**：[日期范围]
   **分析的会议**：[X 次会议]
   **总时长**：[X 小时]
   
   ## 识别的关键模式
   
   ### 1. [主要模式]
   - **观察到**：[你看到的]
   - **影响**：[为什么重要]
   - **建议**：[如何改进]
   
   ### 2. [第二个模式]
   [相同结构]
   
   ## 沟通优势
   
   1. [带示例的优势 1]
   2. [带示例的优势 2]
   3. [带示例的优势 3]
   
   ## 成长机会
   
   1. **[领域 1]**：[具体、可操作的建议]
   2. **[领域 2]**：[具体、可操作的建议]
   3. **[领域 3]**：[具体、可操作的建议]
   
   ## 说话统计
   
   - 平均说话时间：[会议的 X%]
   - 提出的问题：[每次会议平均 X 个]
   - 填充词：[每分钟 X 个]
   - 打断：[每次会议 X 次发出 / Y 次接收]
   
   ## 下一步
   
   [3-5 个改善沟通的具体行动]
   ```

6. **提供后续选项**
   - 在未来会议中追踪相同的指标
   - 深入探讨特定会议或模式
   - 与行业基准比较
   - 创建个人沟通发展计划
   - 为绩效审查生成摘要

## 示例

### 示例 1：冲突回避分析（灵感来自 Dan Shipper）

**用户**："我下载所有会议录音并把它们放在一个文件夹中。告诉我所有我微妙回避冲突的时刻。"

**输出**：
```markdown
# 冲突回避模式

在 15 次会议中发现 23 个你使用间接沟通或未直接解决紧张的情况。

## 模式：对关键反馈模糊

**频率**：7 次会议中 8 次

**示例 1：与 Sarah 的 1:1** - 00:14:32

**发生了什么**：
> "所以，我在想...也许我们可以，像，潜在地
> 考虑再次查看时间线？我的意思是，如果你
> 认为那有意义。但你觉得最好的就行！"

**为什么这很重要**：
你需要解决 Sarah 的项目落后于计划的问题，
但模糊语言（"也许"、"潜在地"、"如果你认为"）
和结尾的推诿（"你觉得最好的就行"）使她容易
错过紧迫性。

**更好的方法**：
"Sarah，项目已经落后两周了。我们需要
讨论什么阻碍了进展并今天创建新的时间线。"

[其他示例...]

## 建议

1. **直接命名问题**在第一句话中
2. **删除模糊词**如"也许"、"有点"、" sort of"
3. **提出具体问题**而非暗示
4. **安排困难对话**而非随意提出
```

### 示例 2：领导引导

**用户**："分析我的团队会议并告诉我关于我的引导风格。"

**输出**：提供关于以下方面的洞察：
- 你说话与团队成员的比例（60% vs. 40%）
- 你是提问还是做陈述（3:1 比例）
- 你如何处理分歧（倾向于过快解决）
- 谁说话最少以及你是否吸引他们
- 良好和错过的引导时刻示例

### 示例 3：个人发展追踪

**用户**："比较我 Q1 与 Q2 的会议，看看我的倾听技能是否改善。"

**输出**：创建比较分析显示：
- 打断减少（每次会议 8 次 → 3 次）
- 澄清问题增加（每次会议 2 → 7 次）
- 在他人想法基础上构建的改进
- 显示差异的具体示例
- 剩余的成長领域

## 设置技巧

### 获取会议转录

**来自 Granola**（Lenny 新闻通讯订阅免费）：
- Granola 自动转录你的会议
- 将转录导出到文件夹：[如何操作的说明]
- 将 Claude Code 指向该文件夹

**来自 Zoom**：
- 启用带转录的云录制
- 会议后下载 VTT 或 SRT 文件
- 存储到专用文件夹

**来自 Google Meet**：
- 使用 Google Docs 自动转录
- 将转录文档保存到文件夹
- 下载为 .txt 文件或给 Claude Code 访问权限

**来自 Fireflies.ai、Otter.ai 等**：
- 批量导出转录
- 存储到本地文件夹
- 对文件夹运行分析

### 最佳实践

1. **一致的命名**：使用 `YYYY-MM-DD - 会议名称.txt` 格式
2. **定期分析**：每月或每季度审查趋势
3. **具体查询**：一次询问一个行为以获得深度
4. **隐私**：将敏感会议数据保留在本地
5. **以行动为导向**：一次专注于一个改进领域
