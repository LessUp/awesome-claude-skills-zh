# 会议洞察分析器

此技能将会议转录转化为关于沟通模式的可行洞察，帮助你成为更有效的沟通者和领导者。

## 何时使用此技能

- 分析多次会议中的沟通模式
- 获取关于领导和引导风格的反馈
- 识别何时回避困难对话
- 了解说话习惯和填充词
- 追踪沟通技能随时间的改进
- 准备带有具体示例的绩效评估
- 指导团队成员的沟通风格

## 此技能的功能

1. **模式识别**：识别跨会议的重复行为，如：
   - 回避冲突或间接沟通
   - 说话比例和轮流发言
   - 提问与陈述的模式
   - 积极倾听指标
   - 决策方法

2. **沟通分析**：评估沟通效果：
   - 清晰度和直接性
   - 填充词和模糊语言的使用
   - 语调和情绪模式
   - 会议控制和引导

3. **可行反馈**：提供带有时间戳的具体示例：
   - 发生了什么
   - 为什么重要
   - 如何改进

4. **趋势追踪**：分析多次会议时比较随时间变化的模式

## 如何使用

### 基本设置

1. 将会议转录下载到文件夹（例如 `~/meetings/`）
2. 在 Claude Code 中导航到该文件夹
3. 请求所需的分析

### 快速入门示例

```
分析此文件夹中的所有会议，告诉我何时回避了冲突。
```

```
查看过去一个月的会议，识别我的沟通模式。
```

```
比较这两个会议文件夹之间的引导风格。
```

### 高级分析

```
分析此文件夹中的所有转录并：
1. 识别何时打断他人
2. 计算我的说话比例
3. 找出回避直接反馈的时刻
4. 追踪填充词的使用
5. 展示良好积极倾听的示例
```

## 说明

当用户请求会议分析时：

1. **发现可用数据**
   - 扫描文件夹中的转录文件（.txt, .md, .vtt, .srt, .docx）
   - 检查文件是否包含说话者标签和时间戳
   - 确认会议的日期范围
   - 识别转录中用户的名称/标识符

2. **明确分析目标**
   
   如果未指定，询问他们想了解什么：
   - 特定行为（回避冲突、打断、填充词）
   - 沟通效果（清晰度、直接性、倾听）
   - 会议引导技能
   - 说话模式和比例
   - 需要改进的成长领域
   
3. **分析模式**

   对于每个请求的洞察：
   
   **回避冲突**：
   - 寻找模糊语言（"也许"、"有点"、"我认为"）
   - 使用间接措辞而非直接请求
   - 紧张出现时改变话题
   - 同意但不承诺（"是的，但是..."）
   - 不解决明显问题
   
   **说话比例**：
   - 计算会议中说话时间的百分比
   - 计算打断次数（用户打断他人和被他人打断）
   - 测量平均发言时长
   - 追踪问题与陈述的比例
   
   **填充词**：
   - 计算"嗯"、"呃"、"就像"、"你知道"、"其实"等的数量
   - 注意每分钟或每次发言的频率
   - 识别增加的情况（紧张、不确定）
   
   **积极倾听**：
   - 引用他人先前观点的问题
   - 转述或总结他人想法
   - 基于他人贡献继续发挥
   - 提出澄清问题
   
   **领导力和引导**：
   - 决策方法（指令式 vs. 协作式）
   - 如何处理分歧
   - 对安静参与者的包容
   - 时间管理和议程控制
   - 跟进和行动项目的清晰度

4. **提供具体示例**

   对于发现的每个模式，包括：
   
   ```markdown
   ### [模式名称]
   
   **发现**：[模式的一句话总结]
   
   **频率**：[Y 次会议中的 X 次]
   
   **示例**：
   
   1. **[会议名称/日期]** - [时间戳]
      
      **发生的情况**：
      > [转录中的实际引用]
      
      **为什么重要**：
      [影响或错失机会的解释]
      
      **更好的方法**：
      [具体的替代措辞或行为]
   
   [重复 2-3 个最有力的示例]
   ```

5. **综合洞察**

   分析所有模式后，提供：
   
   ```markdown
   # 会议洞察总结
   
   **分析期间**：[日期范围]
   **分析会议**：[X 次会议]
   **总时长**：[X 小时]
   
   ## 识别的关键模式
   
   ### 1. [主要模式]
   - **观察到**：[看到的内容]
   - **影响**：[为什么重要]
   - **建议**：[如何改进]
   
   ### 2. [次要模式]
   [相同结构]
   
   ## 沟通优势
   
   1. [优势 1 及示例]
   2. [优势 2 及示例]
   3. [优势 3 及示例]
   
   ## 成长机会
   
   1. **[领域 1]**：[具体、可操作的建议]
   2. **[领域 2]**：[具体、可操作的建议]
   3. **[领域 3]**：[具体、可操作的建议]
   
   ## 说话统计
   
   - 平均说话时间：[会议的 X%]
   - 提出的问题：[平均每次会议 X 个]
   - 填充词：[每分钟 X 个]
   - 打断：[每次会议 X 次主动 / Y 次被动]
   
   ## 下一步
   
   [3-5 个改善沟通的具体行动]
   ```

6. **提供后续选项**
   - 在未来的会议中追踪这些指标
   - 深入研究特定会议或模式
   - 与行业基准进行比较
   - 创建个人沟通发展计划
   - 生成用于绩效评估的总结

## 示例

### 示例 1：回避冲突分析（灵感来自 Dan Shipper）

**用户**："我下载了所有会议录音并放在一个文件夹中。告诉我所有我微妙回避冲突的时候。"

**输出**：
```markdown
# 回避冲突模式

在 15 次会议中发现 23 个使用间接沟通或避免直接解决紧张局势的实例。

## 模式：对关键反馈模糊处理

**频率**：7 次会议中的 8 次

**示例 1：与 Sarah 的 1:1** - 00:14:32

**发生的情况**：
> "所以，我在想...也许我们可以，就像，潜在地
> 考虑再次查看时间线？我的意思是，如果你
> 认为这有意义的话。但无论你认为什么都好！"

**为什么重要**：
你需要解决 Sarah 的项目进度落后的问题，
但模糊语言（"也许"、"潜在地"、"如果你认为"）
和最后的推脱（"无论你认为什么都好"）让她
很容易错过紧迫性。

**更好的方法**：
"Sarah，项目落后两周了。我们需要
今天讨论阻碍进度的问题并创建新的时间线。"

[其他示例...]

## 建议

1. **在第一句话中直接点名问题**
2. **删除模糊词**如"也许"、"有点"、"算是"
3. **提出具体问题**而不是暗示
4. **安排困难对话**而不是随意提出
```

### 示例 2：领导引导

**用户**："分析我的团队会议，告诉我关于我的引导风格。"

**输出**：提供以下洞察：
- 你与团队成员的发言比例（60% vs. 40%）
- 你提问还是陈述（3:1 比例）
- 你如何处理分歧（倾向过快解决）
- 谁发言最少以及你是否引导他们
- 良好和错失的引导时刻示例

### 示例 3：个人发展追踪

**用户**："比较我 Q1 和 Q2 的会议，看看我的倾听技能是否有所改善。"

**输出**：创建比较分析，显示：
- 打断减少（每次会议 8 次 → 3 次）
- 澄清问题增加（每次会议 2 个 → 7 个）
- 基于他人想法继续发挥的改进
- 显示差异的具体示例
- 仍需成长的领域

## 设置提示

### 获取会议转录

**从 Granola**（订阅 Lenny 的新闻通讯免费获取）：
- Granola 自动转录你的会议
- 将转录导出到文件夹：[操作说明]
- 将 Claude Code 指向该文件夹

**从 Zoom**：
- 启用带转录的云端录制
- 会后下载 VTT 或 SRT 文件
- 存储在专用文件夹中

**从 Google Meet**：
- 使用 Google Docs 自动转录
- 将转录文档保存到文件夹
- 下载为 .txt 文件或让 Claude Code 访问

**从 Fireflies.ai、Otter.ai 等**：
- 批量导出转录
- 存储在本地文件夹中
- 对该文件夹运行分析

### 最佳实践

1. **命名一致**：使用 `YYYY-MM-DD - 会议名称.txt` 格式
2. **定期分析**：每月或每季度审查趋势
3. **具体查询**：一次询问一个行为以获得深度
4. **隐私**：保持敏感会议数据本地
5. **行动导向**：一次专注于一个改进领域
