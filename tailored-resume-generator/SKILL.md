# 定制简历生成器

## 何时使用此技能

- 申请特定职位
- 为不同行业或角色定制你的简历
- 为职业转型突出相关经验
- 为 ATS（申请人追踪系统）优化你的简历
- 为不同的求职申请创建多个简历版本
- 强调招聘帖子中提到的特定技能

## 此技能的功能

1. **分析职位描述**：从招聘帖子中提取关键要求、技能、资格和关键词
2. **识别优先级**：根据职位描述的语言和结构确定雇主最看重什么
3. **定制内容**：重新组织并强调相关经验、技能和成就
4. **优化关键词**：在整个简历中自然地融入 ATS 友好的关键词
5. **专业格式化**：创建适合各种格式的干净、专业的简历布局
6. **提供建议**：建议改进并突出需要解决的差距

## 如何使用

### 基础用法
提供职位描述和你的背景信息：

```
我正在申请这份工作：

[粘贴职位描述]

这是我的背景：
- 在 TechCorp 担任 5 年软件工程师
- 带领 3 名开发者的团队进行移动应用项目
- 精通 Python、JavaScript、React
- 州立大学计算机科学学位
```

### 使用现有简历
上传或粘贴你当前的简历以及职位描述：

```
请为这个职位定制我的简历：

职位描述：
[粘贴职位描述]

我当前的简历：
[粘贴简历内容]
```

### 职业转型
当转换行业或角色时：

```
我正在从营销转向产品管理。
这是我正在申请的职位：

[粘贴职位描述]

我的可转移经验：
- 7 年数字营销经验
- 领导跨职能团队
- 管理产品发布
- 数据分析和 A/B 测试
```

## 说明

当用户请求简历定制时：

### 1. 收集信息

**职位描述分析**：
- 如果未提供，请求完整职位描述
- 询问公司名称和职位

**候选人背景**：
- 如果用户提供现有简历，以其为基础
- 如果未提供，请求：
  - 工作历史（职位、公司、日期、职责）
  - 教育背景
  - 关键技能和技术熟练度
  - 显著成就和指标
  - 认证或奖项
  - 任何其他相关信息

### 2. 分析工作要求

提取并优先排序：
- **必备资格**：年经验数、所需技能、教育
- **关键技能**：技术工具、方法论、能力
- **软技能**：沟通、领导力、团队合作
- **行业知识**：领域特定经验
- **关键词**：重复术语、短语和流行语，用于 ATS 优化
- **公司价值观**：来自职位描述的文化契合指标

创建心理地图：
- 优先级 1：关键要求（硬性条件）
- 优先级 2：重要资格（强烈期望）
- 优先级 3：加分技能（额外加分）

### 3. 将候选人经验映射到要求

对于每个工作要求：
- 从候选人背景中识别匹配经验
- 如果没有直接匹配，找到可转移技能
- 注意需要解决或弱化的差距
- 识别要突出的独特优势

### 4. 构建定制简历

**专业摘要**（3-4 行）：
- 以目标角色/领域的年经验数开头
- 包括职位描述中的前 3-4 个所需技能
- 如果相关，提及行业经验
- 突出独特价值主张

**技术/核心技能部分**：
- 按匹配工作要求的类别对技能分组
- 首先列出所需工具和技术
- 使用职位描述中的确切术语
- 仅包含你可以用经验证实的技能

**专业经验**：
- 对于每个角色，强调与工作要求一致的职责和成就
- 使用行动动词：领导、开发、实现、优化、管理、创建、分析
- **量化成就**：包括数字、百分比、时间框架、规模
- 重新排序要点以优先最相关的经验
- 自然地使用职位描述中的关键词
- 格式：**[行动动词] + [什么] + [如何/为什么] + [结果/影响]**

**教育背景**：
- 列出与职位相关的学位、认证
- 如果早期职业，包括相关课程
- 添加匹配工作要求的认证

**可选部分**（如果适用）：
- 认证和执照
- 出版物或演讲
- 奖项和认可
- 志愿工作（如果与角色相关）
- 项目（特别是技术角色）

### 5. 为 ATS（申请人追踪系统）优化

- 使用标准部分标题（专业经验、教育背景、技能）
- 自然地融入职位描述中的确切关键词
- 避免表格、图形、页眉/页脚或复杂格式
- 使用标准字体和要点
- 包括首字母缩写和完整术语（如"SQL（结构化查询语言）"）
- 在真实的情况下匹配职位术语

### 6. 格式化和呈现

**格式选项**：
- **Markdown**：干净、可读、易于复制
- **纯文本**：ATS 优化、适用于所有系统
- **Word/PDF 提示**：提供格式化指导

**简历结构指南**：
- 少于 10 年经验保持 1 页，10 年以上 2 页
- 使用一致的格式和间距
- 确保联系信息突出
- 使用逆时间顺序（最近的在前）
- 保持干净、可扫描的布局，带有空白

### 7. 提供战略建议

在展示定制简历后，提供：

**优势分析**：
- 什么使这个候选人具有竞争力
- 在求职信或面试中强调的独特资格

**差距分析**：
- 未完全满足的要求
- 解决差距的建议（课程、项目、重新框架经验）

**面试准备提示**：
- 与简历对齐的关键话题
- 基于工作要求准备的故事
- 展示契合度的提问

**求职信钩子**：
- 建议 2-3 个求职信开头
- 要扩展的关键成就

### 8. 迭代和完善

询问用户是否想要：
- 调整强调或语气
- 添加或删除部分
- 为不同角色生成替代版本
- 创建格式变体（传统与现代）
- 开发特定角色版本（如果申请多个类似职位）

### 9. 遵循的最佳实践

**做**：
- 保持真实准确——绝不伪造经验
- 使用行业标准化术语
- 用具体指标量化成就
- 为特定工作定制每份简历
- 校对语法和一致性
- 保持语言简洁有力

**不做**：
- 包含个人信息（年龄、婚姻状况、照片，除非要求）
- 使用第一人称代词（我、我的）
- 包含推荐人（"可根据要求提供"已过时）
- 如果职业 20+ 年列出每份工作（专注于相关的、最近的经验）
- 使用未经定制的通用模板
- 除非非常高级的角色否则超过 2 页

### 10. 特殊考虑

**职业转换者**：
- 使用功能性或混合简历格式
- 强调可转移技能
- 在摘要中创建引人注目的叙述
- 专注于相关项目和课程

**应届毕业生**：
- 以教育背景开头
- 包括相关课程、项目、实习
- 强调学生组织中的领导力
- 如果 GPA 3.5+ 包括 GPA

**高级执行官**：
- 以执行摘要开头
- 专注于领导和战略影响
- 包括董事会成员资格、演讲
- 强调收入增长、团队建设、愿景

**技术角色**：
- 显著包括技术技能部分
- 列出编程语言、框架、工具
- 包括 GitHub、作品集或项目链接
- 提及方法论（Agile、Scrum 等）

**创意角色**：
- 包括作品集链接
- 突出创意成就和活动
- 提及工具和软件熟练度
- 考虑更有创意的格式（同时保持 ATS 兼容性）

## 获得最佳结果的提示

- **具体**：提供完整的职位描述和详细的背景信息
- **分享指标**：描述你的经验时包括数字、百分比和可量化的成就
- **表明格式偏好**：让技能知道你是否需要 ATS 优化、创意或传统格式
- **提及约束**：分享任何特定要求（页数限制、要包括/排除的部分）
- **迭代**：不要犹豫请求修订或替代方法
- **多次申请**：为不同角色生成单独的定制版本

## 隐私声明

此技能处理你的个人和职业信息以生成定制简历。提交前始终审查输出以确保准确性和适当性。删除或修改你不想与潜在雇主分享的任何信息。
