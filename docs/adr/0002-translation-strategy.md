# ADR-0002: 中文翻译策略

## 状态

已接受

## 背景

本项目是 awesome-claude-skills 的中文翻译版本，需要制定翻译策略以确保：

1. 内容准确性 - 准确传达原意
2. 表达自然性 - 符合中文表达习惯
3. 术语一致性 - 统一专业术语翻译
4. 维护可行性 - 便于同步上游更新
5. 用户体验 - 中文读者易于理解

## 决策

采用**全面翻译**策略，包括：

1. 所有技能内容（SKILL.md）翻译为中文
2. 所有文档页面翻译为中文
3. 保留技术术语的英文原文或提供对照
4. 代码示例中的注释翻译为中文
5. 保留专有名词的英文（如 Claude、GitHub、Jekyll）

## 理由

### 全面翻译的优势

1. **降低阅读门槛** - 中文读者无需英文能力即可使用
2. **提升可访问性** - 扩大潜在用户群体
3. **保持一致性** - 整体体验统一

### 保留英文专有名词的理由

1. **避免混淆** - 技术术语如 "skill"、"frontmatter" 保持原样
2. **便于搜索** - 用户搜索时使用英文关键词
3. **与上游对照** - 方便对比原项目内容

## 翻译规范

### 技术术语处理

| 英文 | 中文翻译 | 备注 |
|------|----------|------|
| Skill | 技能 | 核心概念 |
| Frontmatter | Frontmatter / 前置元数据 | 保持英文或补充说明 |
| Bundle | 捆绑 / 包 | 根据上下文 |
| Workflow | 工作流 | 标准翻译 |
| Template | 模板 | 标准翻译 |

### 保留英文的情况

1. 专有名词：Claude、GitHub、Jekyll、Ruby 等
2. 文件名和目录名：`SKILL.md`、`scripts/` 等
3. 代码示例：命令、代码块
4. URL 和链接

### 写作风格

- 使用祈使/不定式形式："要完成 X，执行 Y"
- 避免过度翻译："install dependencies" → "安装依赖" 而非 "进行依赖项的安装过程"
- 保持简洁自然

## 后果

### 正面

- 中文读者可直接使用，无需翻译工具
- 扩大项目影响力到中文社区
- 保持术语一致性，降低理解成本

### 负面

- 维护成本增加：需同步上游英文更新
- 翻译质量依赖人工审查
- 可能存在翻译滞后

### 风险缓解

- 在 README.md 中注明最后同步时间
- 建立术语表确保翻译一致性
- 使用 GitHub Issues 跟踪翻译问题和更新
