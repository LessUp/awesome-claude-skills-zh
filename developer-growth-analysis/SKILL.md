# 开发者成长分析

此技能通过分析你的 Claude Code 聊天交互并提供揭示优势和成长领域的模式反馈，为你最近的编码工作提供个性化反馈。

## 何时使用此技能

当你想要时使用此技能：
- 从最近的工作了解你的开发模式和习惯
- 识别特定的技术差距或反复出现的挑战
- 发现哪些主题需要更深入的学习
- 获取根据你实际工作模式定制的精选学习资源
- 跟踪你最近项目中的改进领域
- 查找直接解决你正在发展的技能的高质量文章

此技能非常适合想要获得关于成长的 structured 反馈而无需等待代码审查的开发者，以及喜欢来自自己工作历史的数据驱动洞察的人。

## 此技能的功能

此技能对你的开发工作执行六步分析：

1. **读取你的聊天历史**：访问你过去 24-48 小时的本地 Claude Code 聊天历史，了解你一直在做什么。

2. **识别开发模式**：分析你正在解决的问题类型、使用的技术、遇到的挑战，以及你如何处理不同类型的任务。

3. **检测改进领域**：识别表明技能差距、反复困难、低效方法或可能需要更深入知识的领域的模式。

4. **生成个性化报告**：创建综合报告，显示你的工作摘要、识别的改进领域和具体的成长建议。

5. **查找学习资源**：使用 HackerNews 精选与你的改进领域直接相关的高质量文章和讨论，为你提供根据实际开发工作定制的阅读列表。

6. **发送到你的 Slack 私信**：自动将完整报告发送到你自己的 Slack 直接消息，以便你可以随时随地参考。

## 如何使用

让 Claude 分析你最近的编码工作：

```
从我的最近聊天分析我的开发者成长
```

或者更具体地说明时间段：

```
分析我今天的工作并建议改进领域
```

该技能将生成格式化的报告，包含：
- 你最近工作的概览
- 识别的关键改进领域
- 每个领域的具体建议
- 来自 HackerNews 的精选学习资源
- 你可以专注的行动项

## 说明

当用户请求分析他们最近工作的开发者成长或编码模式时：

1. **访问聊天历史**

   从 `~/.claude/history.jsonl` 读取聊天历史。此文件是 JSONL 格式，每行包含：
   - `display`：用户的消息/请求
   - `project`：正在处理的项目
   - `timestamp`：Unix 时间戳（毫秒）
   - `pastedContents`：任何粘贴的代码或内容

   根据当前时间戳筛选过去 24-48 小时的条目。

2. **分析工作模式**

   从筛选的聊天中提取并分析以下内容：
   - **项目和领域**：用户在处理什么类型的项目？（例如，后端、前端、DevOps、数据等）
   - **使用的技术**：对话中出现什么语言、框架和工具？
   - **问题类型**：正在解决什么类别的问题？（例如，性能优化、调试、功能实现、重构、设置/配置）
   - **遇到的挑战**：用户在什么问题上遇到困难？查找：
     - 关于类似主题的重复问题
     - 需要多次尝试才能解决的问题
     - 表明知识差距的问题
     - 复杂的架构决策
   - **方法模式**：用户如何解决问题？（例如，有条理、探索性、实验性）

3. **识别改进领域**

   基于分析，识别 3-5 个用户可以改进的具体领域。这些应该是：
   - **具体的**（不是模糊的"改进编码技能"）
   - **基于证据的**（基于实际聊天历史）
   - **可操作的**（可以进行的实际改进）
   - **优先级的**（影响最大的优先）

   好的改进领域示例：
   - "高级 TypeScript 模式（泛型、工具类型、类型守卫）- 你在 [特定项目] 中类型安全方面遇到困难"
   - "错误处理和验证 - 我注意到你修复了几个与缺少 null 检查相关的 bug"
   - "Async/await 模式 - 你最近的工作显示一些竞态条件和时序问题"
   - "数据库查询优化 - 你多次重写同一个查询"

4. **生成报告**

   创建具有此结构的综合报告：

   ```markdown
   # 你的开发者成长报告

   **报告期**：[昨天 / 今天 / [自定义日期范围]]
   **最后更新**：[当前日期和时间]

   ## 工作摘要

   [2-3 段总结用户做了什么、接触的项目、使用的技术和整体重点领域]

   示例：
   "在过去 24 小时中，你主要专注于后端开发，涉及三个不同的项目。你的工作涉及 TypeScript、React 和部署基础设施。你处理了功能实现、调试和架构决策的组合，特别关注 API 设计和数据库优化。"

   ## 改进领域（优先级排序）

   ### 1. [领域名称]

   **为什么这很重要**：[解释为什么此技能对用户工作很重要]

   **我观察到的**：[聊天历史中显示此差距的具体证据]

   **建议**：[在该领域改进的具体步骤]

   **技能提升时间**：[所需努力的简要估计]

   ---

   [重复 2-4 个额外领域]

   ## 观察到的优势

   [2-3 个要点强调你做得好的事情 - 要继续做的事情]

   ## 行动项

   优先级顺序：
   1. [从最高优先级改进领域衍生的行动项]
   2. [下一个领域的行动项]
   3. [下一个领域的行动项]

   ## 学习资源

   [将在下一步填充]
   ```

5. **搜索学习资源**

   使用 Rube MCP 在 HackerNews 上搜索与每个改进领域相关的文章：

   - 对于每个改进领域，构建针对高质量资源的搜索查询
   - 使用 RUBE_SEARCH_TOOLS 搜索 HackerNews，查询如：
     - "学习 [技术/模式] 最佳实践"
     - "[技术] 高级模式和技术"
     - "在 [语言] 中调试 [特定问题类型]"
   - 优先考虑高参与度的帖子（评论、点赞）
   - 对于每个领域，包含 2-3 篇最相关的文章，包括：
     - 文章标题
     - 发布日期
     - 为什么相关的简要描述
     - 文章链接

   将此部分添加到报告：

   ```markdown
   ## 精选学习资源

   ### 针对：[改进领域]

   1. **[文章标题]** - [日期]
      [涵盖内容以及为什么与你的改进领域相关的描述]
      [链接]

   2. **[文章标题]** - [日期]
      [描述]
      [链接]

   [重复其他改进领域]
   ```

6. **展示完整报告**

   以清晰、可读的格式提供报告，用户可以：
   - 快速浏览关键要点
   - 用于聚焦的学习规划
   - 在接下来的一周改进时参考
   - 如果想要外部反馈，与导师分享

7. **将报告发送到 Slack 私信**

   使用 Rube MCP 将完整报告发送到用户自己的 Slack 私信：

   - 通过 RUBE_SEARCH_TOOLS 检查 Slack 连接是否活跃
   - 如果未连接，使用 RUBE_MANAGE_CONNECTIONS 启动 Slack 认证
   - 使用 RUBE_MULTI_EXECUTE_TOOL 将报告作为格式化消息发送：
     - 将报告标题和期限作为第一条消息发送
     - 将报告分解为逻辑部分（摘要、改进、优势、行动、资源）
     - 将每个部分格式化为结构良好的 Slack 消息，使用正确的 markdown
     - 包含学习资源的可点击链接
   - 在 CLI 输出中确认发送

   这确保用户在经常查看的地方拥有报告，并可以在整周内参考。

## 改进领域（优先级排序）

### 1. 高级 TypeScript 模式和类型安全

**为什么这很重要**：TypeScript 是你工作的核心，但利用其高级功能（泛型、工具类型、条件类型、类型守卫）可以显著提高代码可靠性并减少运行时错误。更好的类型安全在编译时而非生产中捕获 bug。

**我观察到的**：在你最近的聊天中，你在处理连接数据结构，几次在正确输入 auth 配置方面遇到困难。你还必须迭代不同连接状态的联合类型。有机会更有效地使用可辨识联合和类型守卫。

**建议**：学习 TypeScript 的高级类型系统，特别是工具类型（Omit、Pick、Record）、条件类型和可辨识联合。将这些模式应用于你的连接配置处理和 auth 状态管理。

**技能提升时间**：5-8 小时的聚焦学习和实践

### 2. 安全数据处理和 UI 信息隐藏

**为什么这很重要**：你识别并修复了一个安全问题，敏感连接数据在你的控制台中显示。防止信息泄露对于处理用户凭证和 API 密钥的应用程序至关重要。这里的良好实践可以防止安全事件和用户信任违规。

**我观察到的**：你发现"你的应用"页面显示完整的连接数据，包括 auth 配置。这显示了良好的安全意识，下一步是在处理敏感信息时将其建立为默认思维。

**建议**：审查在前端应用中处理敏感数据的安全最佳实践。创建可重用模式，在显示前过滤/屏蔽敏感信息。考虑实现一个安全数据层，明确白名单可以在 UI 中显示的内容。

**技能提升时间**：3-4 小时

### 3. 组件架构和响应式 UI 模式

**为什么这很重要**：你正在设计需要在不同屏幕尺寸和用户交互下工作的 UI。强大的组件架构使构建复杂 UI 更容易且无 bug，并提高可维护性。

**我观察到的**：你开发了"市场"UI（原浏览工具），从设计图像重新创建。你还识别并修复了内容溢出容器的滚动问题。有机会加强对布局包含和响应式设计模式的理解。

**建议**：学习 React 组件组合模式和 CSS 布局最佳实践（特别是 flexbox 和 grid）。专注于容器查询和防止溢出问题的响应式模式。研究组件组合库和设计系统方法。

**技能提升时间**：6-10 小时（取决于深度）

## 观察到的优势

- **安全意识**：你在数据泄露问题成为问题之前主动识别
- **迭代完善**：你有条理地处理 UI 需求，提出澄清问题并改进设计
- **全栈能力**：你能舒适地处理后端 API、前端 UI 和部署问题
- **问题解决方法**：你将复杂任务分解为可管理的步骤

## 行动项

优先级顺序：
1. 花 1-2 小时学习 TypeScript 工具类型和可辨识联合；应用于你的连接数据结构
2. 为你的项目记录安全模式（哪些数据可以安全显示、过滤/屏蔽函数）
3. 学习一篇关于高级 React 模式的文章并将一个模式应用于你当前的 UI 工作
4. 设置专注于类型安全和数据安全的代码审查检查清单，用于未来的 PR

## 提示和最佳实践

- 每周运行一次此分析以跟踪你的改进轨迹随时间的变化
- 一次选择一个改进领域并专注几天，然后再转向下一个
- 将学习资源作为学习指南；通读推荐材料并练习应用模式
- 在专注于某个领域一周后重新访问此报告，看看你的工作模式如何变化
- 学习资源是有意为你的实际工作精选的，而非通用主题，因此它们将与你正在构建的内容高度相关

## 如何保持准确性和质量

此技能：
- 分析来自时间戳聊天历史的实际工作模式
- 生成基于证据的建议，基于真实项目
- 精选直接解决你识别差距的学习资源
- 专注于可操作的改进，而非模糊的反馈
- 根据复杂性提供具体的时间估计
- 优先考虑对开发速度影响最大的领域
